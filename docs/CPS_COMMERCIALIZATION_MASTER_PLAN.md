# AIdeal CPS 商用落地总方案（2026-04 刷新稿）

## 1. 总目标
将 AIdeal CPS 从“能跑的公众号导购原型”升级为“可持续运营的商用微信公众号导购系统”。

## 2. 商用落地范围
### 2.1 前台链路
- 微信服务号菜单交互
- 找商品
- 今日推荐
- 合伙人中心
- 图文详情页
- 下单跳转页
- 更多同类产品入口
- 后续个性化推荐

### 2.2 后台链路
- 京东商品刷新
- 短链刷新
- 商品评分/排序
- 商品合规过滤
- 曝光记录
- 点击归因
- 用户画像
- 合伙人归因

### 2.3 工程链路
- 新加坡开发环境
- GitHub 文档记忆
- Codex CLI 受控开发
- 杭州生产环境自动部署
- 回滚与 smoke test 机制

## 3. 当前已知稳定能力
- 微信服务号消息推送已打通
- /wechat/callback 为真实稳定回调路径
- jingfen.query 可稳定取商品
- promotion.bysubunionid.get 可稳定转短链
- H5 图文详情链路已具备基础能力
- redirect 已具备 302 到 JD short_url 的基础能力

## 4. 当前未完成/待收口问题
- 推荐/H5/redirect 代码曾被多轮修补，需做架构收口
- 推荐服务存在重复实现/覆盖式修复历史
- 商品刷新与推荐池构建需要重新定义主链路与 fallback 策略
- 开发机到生产机自动部署尚未建完
- 免费模型驱动的模板优化与AB测试尚未工程化落地

## 5. 商用原则
1. 图优先
2. 路径最短
3. 推荐真实
4. 文案可解释
5. 配置优先
6. 架构稳定优先于临时可用
7. GitHub docs 是长期记忆，不依赖聊天短期上下文

## 6. 推荐系统原则
- 商品推荐不是只看佣金
- 必须综合：
  - 真实优惠力度
  - 销量/成交
  - 评论量
  - 好评率
  - 店铺质量
  - 品类多样性
  - 用户偏好
- 推荐理由可以利用心理学，但不能伪造事实
- 当前轮次内不重复，池耗尽再轮转
- 同轮次内避免高度相似商品挤占 3 个坑位

## 7. 免费模型使用原则
- 免费模型只用于：
  - 文案AB变体生成
  - 推荐理由风格增强
  - 规则优化候选建议
- 免费模型不直接决定事实字段
- 事实字段只能来自京东真实数据或系统真实行为数据

## 8. 当前执行顺序
### P0
1. 稳定开发机规范
2. 稳定推荐/H5/redirect 架构
3. 稳定商品刷新主链路
4. 建立部署链路
5. 形成商用 smoke test 套件

### P1
1. 微信图优先消息完善
2. 更多同类产品升级成公众号内返回
3. 用户画像驱动推荐
4. 免费模型做文案优化/AB测试
5. 后台运营配置化

## 9. Do / Don't
### Do
- 先审计再修改
- 先配置化再编码
- 先开发机验证再部署生产
- 每次更新 docs
- 每次提交说明边界与风险

### Don't
- 不要在生产机直接堆临时补丁
- 不要在一个文件里重复定义函数
- 不要把业务常量散落硬编码
- 不要把“不知道为什么能跑”的代码带进商用链路
