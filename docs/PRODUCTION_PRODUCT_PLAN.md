# 智省优选商用产品设计（V1）

## 1. 产品定位
智省优选不是普通卖货号，而是：
- 理性消费导向的 AI 导购顾问
- 基于京东联盟的商品池与转链能力
- 以“更值、更省、更靠谱”为核心体验

## 2. 核心原则
1. 用户价值优先，商业价值第二
2. 在不损害用户体验的前提下最大化佣金
3. 不推荐高风险商家，除非是独家供给
4. 推荐系统先过滤，再排序，再生成话术
5. 商品推荐优先消费本地商品池，必要时再补查京东
6. 所有链路都要为点击率、下单率、复购率服务

## 3. 用户主流程
1. 关注公众号
2. 收到欢迎语与使用引导
3. 用户自然语言表达购买需求
4. 系统做意图识别
5. 从本地商品池召回候选
6. 结合商品评分 + 商家画像 + 商业价值排序
7. 返回 3 个商品建议
8. 点击自有跳转链接后 302 到 JD 短链
9. 订单回流后更新用户画像、商品表现、商家画像
10. 次日 8:00 做个性化推荐

## 4. 推荐三件商品的固定角色
- 商品 1：最符合用户需求的稳妥款
- 商品 2：销量 / 口碑最强款
- 商品 3：性价比冲击款

## 5. 排序三分系统
### 5.1 用户价值分
- 价格匹配度
- 券后价
- 评论量
- 好评率
- 自营 / 官方旗舰店
- 用户偏好匹配

### 5.2 商家健康分
- 店铺评分
- 售后评分
- 物流 / 履约评分
- 价格虚高风险
- 历史投诉 / 差评 / 退货风险（当前先用代理指标）

### 5.3 商业价值分
- 佣金率
- 预估佣金
- 转链成功率
- 券 / 链接稳定性

### 5.4 初始权重建议
- 用户价值分 50%
- 商家健康分 30%
- 商业价值分 20%

## 6. 商家画像
### 6.1 一期目标
建立 merchant_profiles 表，记录：
- shop_id
- shop_name
- 店铺评分代理字段
- merchant_health_score
- risk_flags
- recommendable

### 6.2 当前代理信号
由于当前链路直接可取到的京东字段有限，一期先使用：
- userEvaluateScore
- afterServiceScore
- logisticsLvyueScore
- scoreRankRate
- shopLabel
- owner
- 同批次价格中位数偏离度

### 6.3 风险标签
- poor_reputation
- poor_after_sales
- poor_fulfillment
- price_too_high

### 6.4 推荐规则
- 默认只推荐 recommendable=True 的商品
- 非 recommendable 商品保留在后台池中供人工查看
- 除非独家供给，否则高风险商家不推给用户

2. 欢迎流与导购引导
3. 三商品推荐引擎
4. 自有点击归因跳转网关
5. 多榜单定时同步
6. 商品池优先驱动公众号问答

## 8. 当前 P1
1. 8:00 个性化推荐
2. 海报卡生成
3. 用户画像打分
4. A/B 测试


## 9. 当前一期商家画像策略（2026-04）
- 先用中性基线分，不因缺少字段自动判为高风险
- 自营 / 官方旗舰店给正向加权
- price_too_high 与低口碑/低售后组合时可直接降为不可推荐
- /products 默认只返回 merchant_recommendable=true 的商品


## 10. 公众号欢迎流与三商品推荐（V1）
- 欢迎语不做说明书式介绍，直接引导用户说购买需求
- 默认返回 3 个候选：
  1. 最符合需求
  2. 下单热度更高
  3. 口碑与店铺更稳
- 第一版优先使用数据库商品池与商家画像过滤
- 自然语言推荐先采用规则解析，后续再接模型增强


## 11. 公众号文案配置管理（V1）
- 运营文案先放配置文件，不写死在业务代码
- 当前文件：config/wechat_dialog_copy.json
- 适用对象：
  - 欢迎语
  - 帮助文案
  - 非购物回转文案
  - 无命中兜底文案


## 12. 点击归因跳转网关（V1）
- 对外链接统一使用自有 redirect 链接
- redirect 时先写 click_logs，再 302 到 JD 短链
- 第一版记录：
  - wechat_openid
  - product_id
  - subunionid
  - scene
  - slot
  - trace_id
  - final_url
  - user_agent
  - referer


## 13. 用户画像与每日推荐（V1）
- 用户画像先做基础偏好分：
  - price_sensitive_score
  - quality_sensitive_score
  - sales_sensitive_score
  - self_operated_sensitive_score
- preferred_categories 先存 JSON
- 每日 8:00 先生成推荐候选，不直接下发
- 推荐优先按用户偏好品类与偏好模式排序


## 14. 商品海报卡与晨推作业（V1）
- 海报卡先使用 SVG 生成，不额外引入图片库依赖
- 晨推先输出 job.json 与 poster 文件
- 先做“候选生成与落盘”，后续再接实际消息发送
- cron 接入脚本先提供，不立即执行


## 15. 海报卡视觉统一原则（V1）
- 海报卡视觉优先对齐官网，而不是做成促销风长图
- 结构遵循：
  - 品牌抬头
  - 商品标题
  - 到手价
  - 理由
  - 查看动作
- 风格关键词：
  - 理性
  - 克制
  - 干净
  - 可信


## 16. 合伙人系统设计原则（V1）
- 合伙人首先是用户，所以一键购买永远放在第一位
- 分享分发能力是第二位
- 收益展示与等级体系是第三位
- 先做归因与积分账本基础，不做现金提现


## 17. 合伙人资产包设计（V1）
- 一键购买链接放第一优先级
- 分享链接与分享文案放第二优先级
- 产出二维码 SVG、品牌海报 SVG
- 京口令先预留字段，后续正式接入


## 18. 合伙人收益账本（V1）
- 账本事件：
  - estimated
  - settled
  - reversed
  - redeem
  - adjustment
- 概览输出：
  - available_points
  - estimated_reward
  - settled_reward
  - reversed_reward
  - redeemed_points
- 等级升级先按净已结算佣金自动判定


## 19. 积分消耗闭环（V1）
- 积分不能只发不花，前台必须明确展示可消费场景
- 当前先定义三类消耗：
  - 合伙人开通/续费抵扣
  - 精选选品服务包抵扣
  - 推广海报与素材包抵扣
- 当前不支持现金提现，避免合规与投诉风险失控


## 20. 积分消耗最小闭环（V1）
- 100元合伙人开通/续费商品化
- 支持积分全额或部分抵扣
- 当前先记录 pending_payment / completed，不直接做现金支付
- 前台必须可查看：
  - 可消费项
  - 抵扣试算
  - 消耗历史
  - 当前是否已完成开通门槛


## 21. 合伙人中心最小可用版（V1）
- 一个聚合接口直接供前台取数
- 展示：
  - 我的等级
  - 我的积分
  - 可消费项
  - 消耗历史
  - 最近可分享商品
  - 最近已生成分享资产
- 前台优先引导：
  - 一键购买
  - 分享分发
  - 积分消耗


## 22. 合规治理与年龄分流（V1）
- 硬禁推：违法违禁、武器化防身、明显高风险商品
- 限制级：默认不主动推荐，不做合伙人分享
- 未校验用户：仅可见 normal
- adult_verified 用户：可见 normal + restricted
- 公众号默认分发面和晨推面只允许 normal


## 23. adult_verified 最小闭环（V1）
- 用户通过 H5 完成成年声明
- 系统写入 adult_verified / adult_verified_at / verification_source
- restricted 商品仅允许被动检索可见
- 公众号默认推荐、晨推、合伙人分享仍只允许 normal


## 24. 实名 / 年龄核验后置规划
- 当前 self_declaration_h5 仅作占位，不作为 restricted 商品正式放开的唯一依据
- 后续优先评估国家网络身份认证公共服务
- 次选阿里云实人认证
- 若接入实名能力，默认先用于年龄闸门，不直接改动 normal 主链路


## 25. 真实搜索可用升级（V1）
- 本地商品池不足时，自动回退到 JD 实时关键词搜索
- 默认返回 3 个可买链接
- 仍沿用商品合规过滤
- 不依赖先把所有品类预灌入数据库


## 26. 商品池夜间刷新与有效期治理（规划）
- 白天：优先命中本地商品池，保证速度和稳定性
- 夜间低负载时：跑后台刷新任务，完成商品补齐、排序、打分、短链补全
- 按有效期与最后同步时间做清理、降权、失效下架
- 实时关键词搜索仅作为长尾需求补充，不应长期替代主商品池


## 27. 夜间商品池刷新任务骨架（V1）
- 夜间刷新：
  - 精选榜商品
  - 关键词种子商品
- 写回 last_sync_at，持续补池
- 超过 expire_hours 的商品自动 inactive
- 超过 purge_hours 且无下游引用的 inactive 商品自动清理
- 白天用户查询优先命中本地池


## 28. 夜间刷新调度（V1）
- 调度方式：deploy 用户 crontab
- 执行时间：每日 03:15
- 执行脚本：scripts/run_nightly_catalog_refresh.py
- 防重入：flock /tmp/aideal_catalog_refresh.lock
- 日志文件：logs/catalog_refresh.log
- 当前策略：精选榜刷新为主，关键词补池为辅
