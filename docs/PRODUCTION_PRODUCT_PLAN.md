# 智省优选商用产品设计（V1）

## 1. 产品定位
智省优选不是普通卖货号，而是：
- 理性消费导向的 AI 导购顾问
- 基于京东联盟的商品池与转链能力
- 以“更值、更省、更靠谱”为核心体验

## 2. 核心原则
1. 用户价值优先，商业价值第二
2. 在不损害用户体验的前提下最大化佣金
3. 不推荐高风险商家，除非是独家供给
4. 推荐系统先过滤，再排序，再生成话术
5. 商品推荐优先消费本地商品池，必要时再补查京东
6. 所有链路都要为点击率、下单率、复购率服务

## 3. 用户主流程
1. 关注公众号
2. 收到欢迎语与使用引导
3. 用户自然语言表达购买需求
4. 系统做意图识别
5. 从本地商品池召回候选
6. 结合商品评分 + 商家画像 + 商业价值排序
7. 返回 3 个商品建议
8. 点击自有跳转链接后 302 到 JD 短链
9. 订单回流后更新用户画像、商品表现、商家画像
10. 次日 8:00 做个性化推荐

## 4. 推荐三件商品的固定角色
- 商品 1：最符合用户需求的稳妥款
- 商品 2：销量 / 口碑最强款
- 商品 3：性价比冲击款

## 5. 排序三分系统
### 5.1 用户价值分
- 价格匹配度
- 券后价
- 评论量
- 好评率
- 自营 / 官方旗舰店
- 用户偏好匹配

### 5.2 商家健康分
- 店铺评分
- 售后评分
- 物流 / 履约评分
- 价格虚高风险
- 历史投诉 / 差评 / 退货风险（当前先用代理指标）

### 5.3 商业价值分
- 佣金率
- 预估佣金
- 转链成功率
- 券 / 链接稳定性

### 5.4 初始权重建议
- 用户价值分 50%
- 商家健康分 30%
- 商业价值分 20%

## 6. 商家画像
### 6.1 一期目标
建立 merchant_profiles 表，记录：
- shop_id
- shop_name
- 店铺评分代理字段
- merchant_health_score
- risk_flags
- recommendable

### 6.2 当前代理信号
由于当前链路直接可取到的京东字段有限，一期先使用：
- userEvaluateScore
- afterServiceScore
- logisticsLvyueScore
- scoreRankRate
- shopLabel
- owner
- 同批次价格中位数偏离度

### 6.3 风险标签
- poor_reputation
- poor_after_sales
- poor_fulfillment
- price_too_high

### 6.4 推荐规则
- 默认只推荐 recommendable=True 的商品
- 非 recommendable 商品保留在后台池中供人工查看
- 除非独家供给，否则高风险商家不推给用户

## 9. 当前一期商家画像策略（2026-04）
- 先用中性基线分，不因缺少字段自动判为高风险
- 自营 / 官方旗舰店给正向加权
- price_too_high 与低口碑/低售后组合时可直接降为不可推荐
- /products 默认只返回 merchant_recommendable=true 的商品


## 10. 公众号欢迎流与三商品推荐（V1）
- 欢迎语不做说明书式介绍，直接引导用户说购买需求
- 默认返回 3 个候选：
  1. 最符合需求
  2. 下单热度更高
  3. 口碑与店铺更稳
- 第一版优先使用数据库商品池与商家画像过滤
- 自然语言推荐先采用规则解析，后续再接模型增强


## 11. 公众号文案配置管理（V1）
- 运营文案先放配置文件，不写死在业务代码
- 当前文件：config/wechat_dialog_copy.json
- 适用对象：
  - 欢迎语
  - 帮助文案
  - 非购物回转文案
  - 无命中兜底文案


## 12. 点击归因跳转网关（V1）
- 对外链接统一使用自有 redirect 链接
- redirect 时先写 click_logs，再 302 到 JD 短链
- 第一版记录：
  - wechat_openid
  - product_id
  - subunionid
  - scene
  - slot
  - trace_id
  - final_url
  - user_agent
  - referer


## 13. 用户画像与每日推荐（V1）
- 用户画像先做基础偏好分：
  - price_sensitive_score
  - quality_sensitive_score
  - sales_sensitive_score
  - self_operated_sensitive_score
- preferred_categories 先存 JSON
- 每日 8:00 先生成推荐候选，不直接下发
- 推荐优先按用户偏好品类与偏好模式排序
